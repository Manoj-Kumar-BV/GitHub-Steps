Basic GitHub Commands

Here are some of the most commonly used GitHub commands:

  1. **Clone a Repository**:
    • This command is used to clone a repository to your local machine.
    git clone https://github.com/YourUsername/YourRepository.git
cd YourRepository
  2. **Check the Status of Your Repository**:
    • This command is used to check the status of your files in the working directory and staging area.
    git status
  3. **Stage Changes**:
    • This command is used to stage changes for the next commit.
    git add .
  4. **Commit Changes**:
    • This command is used to commit staged changes with a message.
    git commit -m "Description of changes"
  5. **Push Changes to Remote Repository**:
    • This command is used to push commits from your local repository to the remote repository.
    git push origin main  # or 'master' if your branch is named 'master'
  6. **Pull Latest Changes from Remote Repository**:
    • This command is used to pull the latest changes from the remote repository to your local repository.
    git pull origin main  # or 'master' if your branch is named 'master'
  7. **Create a New Branch**:
    • This command is used to create a new branch.
    git checkout -b feature-name
  8. **Merge a Branch**:
    • This command is used to merge a branch into your current branch.
    git merge feature-name
  9. **Delete a Branch**:
    • This command is used to delete a branch.
    git branch -d feature-name

Next Steps

After learning these basic commands, you can:

  • Create repositories to manage different projects.
  • Create branches to manage different features or versions of your project.
  • Start committing code to your repositories.
  • Collaborate with others by inviting them to your repository.
